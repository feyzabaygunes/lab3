Stringler

JavaScript dizeleri metin depolamak ve değiştirmek için kullanılır.

JavaScript Dizeleri

JavaScript dizesi, tırnak içine yazılmış sıfır veya daha fazla karakterdir.

var x = "John Doe";

Tek veya çift tırnak kullanabilirsiniz:

var carName1 = "Volvo XC60"; // İkili alıntı

var carName2 = 'Volvo XC60'; // Tek tırnak

Dizeyi çevreleyen tırnak işaretleriyle eşleşmedikleri sürece, bir dizenin içindeki tırnak işaretlerini kullanabilirsiniz:

var answer1 = "Her şey yolunda";

var answer2 = "Ona 'Johnny' deniyor";

var answer3 = 'Ona "Johnny" denir';

Dize uzunluğu

Bir dizenin uzunluğunu bulmak için yerleşik length özelliğini kullanın:

var txt = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";

var sln = txt.length;

Kaçış Karakteri

Dizeler tırnak içinde yazılması gerektiğinden, JavaScript bu dizeyi yanlış anlayacaktır:

var x = "Biz kuzeyden" Vikingler "diyoruz.";

Dize "Biz sözde" olarak kesilecektir.

Bu problemden kaçınmanın çözümü, ters eğik çizgi kaçış karakterini kullanmaktır.

Ters eğik çizgi (\) çıkış karakteri özel karakterleri dize karakterlerine dönüştürür:

Kod Sonuç Açıklama
\' ' Tek alıntı
\" " Çift tırnak
\\ \ Backslash

\ "Dizisi bir dizeye çift tırnak ekler:

var x = "Biz kuzeyden \" Vikingler \ "diyoruz.";


Dizi bir dizeye tek bir tırnak ekler:

var x = 'Tamam.';


\\ dizisi bir dizeye ters eğik çizgi ekler:

var x = "\\ karakterine ters eğik çizgi denir.";


JavaScript'te altı başka kaçış dizisi daha geçerlidir:

Kod Sonucu
\b geri tuşu
\f Form besleme
\n Yeni satır
\r Satırbaşı
\t Yatay Tablo
\v Dikey Tablo

Yukarıdaki 6 kaçış karakteri orijinal olarak daktiloları, teletipleri ve faks makinelerini kontrol etmek için tasarlanmıştır. HTML'de bir anlam ifade etmiyorlar.

Uzun Kod Satırlarını Kırmak

En iyi okunabilirlik için, programcılar genellikle 80 karakterden uzun kod satırlarından kaçınmayı sever.

Bir JavaScript ifadesi bir satıra sığmıyorsa, bunu kırmak için en iyi yer bir operatörden sonradır:

document.getElementById ("demo"). innerHTML = "Merhaba dolly!";

Ayrıca, metin dizesi içindeki bir kod satırını tek bir ters eğik çizgiyle ayırabilirsiniz:

document.getElementById ("demo"). innerHTML = "Merhaba \ Dolly "!;

\ Yöntemi tercih edilen yöntem değildir. Evrensel desteği olmayabilir. Bazı tarayıcılar \ karakterinin arkasındaki boşluklara izin vermez.


Bir dizeyi ayırmanın daha güvenli bir yolu, dize eklemesini kullanmaktır:

document.getElementById ("demo"). innerHTML = "Merhaba" + "Dolly!";


Bir kod satırını ters eğik çizgiyle ayıramazsınız:

document.getElementById ("demo"). innerHTML = \ "Merhaba dolly!";

Dizeler Nesne Olabilir

Normalde, JavaScript dizeleri değişmez değerlerden oluşturulan ilkel değerlerdir:

  • var firstName = "John";

    • Ancak dizeler, new anahtar kelimesine sahip nesneler olarak da tanımlanabilir:

  • var firstName = yeni Dize ("John");

    • var x = "John";

    var y = yeni Dize ("John");


    // typeof x, string döndürür

    // typeof y nesneyi döndürür

    Nesne olarak dizeler oluşturmayın. Yürütme hızını yavaşlatır. Yeni anahtar kelime kodu karmaşık hale getirir. Bu bazı beklenmedik sonuçlar doğurabilir.

    == operatörünü kullanırken, eşit dizeler eşittir:

    var x = "John";

    var y = yeni Dize ("John");


    // (x == y) doğru çünkü x ve y eşit değerlere sahip

    === işleci kullanılırken eşit dizeler eşit değildir, çünkü === işleci hem tür hem de değerde eşitlik bekler.

    var x = "John";

    var y = yeni Dize ("John");


    // (x === y) yanlıştır, çünkü x ve y farklı türlere sahiptir (dize ve nesne)

    Veya daha da kötüsü. Nesneler karşılaştırılamaz:

    var x = yeni Dize ("John");

    var y = yeni Dize ("John");


    // (x == y) yanlış çünkü x ve y farklı nesneler

    (X == y) ve (x === y) arasındaki farka dikkat edin.

    İki JavaScript nesnesinin karşılaştırılması her zaman yanlış döndürür.

    Sayılar

    JavaScript'in yalnızca bir tür numarası vardır. Sayılar ondalıklı veya ondalıksız olarak yazılabilir.

    var x = 3.14; // Ondalık basamaklı bir sayı

    var y = 3; // Ondalıksız sayı


    Çok büyük veya çok küçük sayılar bilimsel (üslü) gösterimle yazılabilir:

    var x = 123e5; // 12300000

    var y = 123e-5; // 0.00123

    JavaScript Numaraları Her Zaman 64 bit Kayan Noktadır

    Diğer birçok programlama dilinin aksine, JavaScript tamsayılar, kısa, uzun, kayan nokta vb. gibi farklı sayı türlerini tanımlamaz.

    JavaScript numaraları her zaman çift kesinlikli kayar nokta sayısı olarak saklanır, uluslararası IEEE 754 standardını izleyerek.

    Bu biçim sayıları 64 bit olarak saklar; burada sayı (kesir) 0 ila 51 bitlerinde, üs 52 ila 62 bitlerinde ve oturum açma biti 63'te saklanır:

    Değer (aka Kesir / Mantissa) Üs İşaret
    52 bit (0 - 51) 11 bit (52 - 62) 1 bit (63)

    Hassaslık

    Tamsayılar (nokta veya üs notasyonu olmayan sayılar) 15 basamağa kadar doğrudur.

    var x = 999999999999999; // x 999999999999999 olacak

    var y = 9999999999999999; // y 10000000000000000 olacak


    Maksimum ondalık sayı 17'dir, ancak kayan nokta aritmetiği her zaman %100 doğru değildir:

    var x = 0.2 + 0.1 'dir; // x 0.30000000000000004 olacak


    Yukarıdaki sorunu çözmek için çoğalmaya ve bölmeye yardımcı olur:

    var x = (0.2 x 10 + 0.1 x 10) / 10; // x 0,3 olacak

    Sayı ve Dize Ekleme

    UYARI !!

    JavaScript hem ekleme hem de birleştirme için + işlecini kullanır.

    Sayılar eklenir.

    Dizeler birleştirilir.

    İki sayı eklerseniz sonuç bir sayı olur:

    var x = 10;

    var y = 20;

    var z = x + y; // z 30 (bir sayı) olacak


    İki dize eklerseniz, sonuç bir dize birleşimi olur:

    var x = "10";

    var y = "20";

    var z = x + y; // z 1020 olacak (bir dize)


    Bir sayı ve dize eklerseniz, sonuç bir dize birleşimi olur:

    var x = 10;

    var y = "20";

    var z = x + y; // z 1020 olacak (bir dize)


    Bir dize ve sayı eklerseniz, sonuç bir dize birleştirmesi olur:

    var x = "10";

    var y = 20;

    var z = x + y;

    // z 1020 olacak (bir dize)


    Yaygın bir hata, bu sonucun 30 olmasını beklemektir:

    var x = 10;

    var y = 20;

    var z = "Sonuç:" + x + y;


    Yaygın bir hata, bu sonucun 102030 olmasını beklemektir:

    var x = 10;

    var y = 20;

    var z = "30";

    var sonuç = x + y + z;


    JavaScript yorumlayıcısı soldan sağa doğru çalışır.

    İlk 10 + 20 eklenir, çünkü x ve y'nin her ikisi de sayıdır.

    Daha sonra z + bir dize olduğu için 30 + "30" birleştirilir.
