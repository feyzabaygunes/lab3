JavaScript Stil Kılavuzu ve Kodlama Kuralları

Tüm JavaScript projeleriniz için her zaman aynı kodlama kurallarını kullanın.

JavaScript Kodlama Kuralları

Kodlama kuralları programlama için stil yönergeleridir. Genellikle şunları kapsar:

  • Değişkenler ve fonksiyonlar için adlandırma ve beyan kuralları.

  • Beyaz boşluk kullanımı için kurallar, girinti ve yorumlar.

  • Programlama uygulamaları ve ilkeleri


    • Kodlama kuralları kaliteyi güvence altına alır:

  • Kod okunabilirliğini artırır

  • Kod bakımını kolaylaştırır

    • Kodlama kuralları, ekiplerin izlemesi gereken kurallar olarak belgelenebilir veya yalnızca bireysel kodlama uygulamanız olabilir.

    Değişken İsimleri

    Tüm isimler bir harfle başlar.

    firstName = "John";

    lastName = "Doe";


    fiyat = 19,90;

    vergi = 0.20;


    fullPrice = fiyat + (fiyat * vergi);

    Operatörlerin Çevresindeki Alanlar

    Her zaman operatörlerin etrafına (= + - * /) ve virgüllerden sonra boşluk koyun:

    var x = y + z;

    var değerler = ["Volvo", "Saab", "Fiat"];

    Kod Girintisi

    Kod bloklarının girintisi için her zaman 2 boşluk kullanın:

    Fonksiyonlar:

    functionCelsius (fahrenheit) {

    return (5/9) * (fahrenhayt - 32);

    }

    Girinti için tırnaklar (tablolar) kullanmayın. Farklı editörler sekmeleri farklı yorumlar.

    Bildirim Kuralları

    Basit ifadeler için genel kurallar:

  • Basit bir ifadeyi her zaman noktalı virgülle bitirin.

    • var değerler = ["Volvo", "Saab", "Fiat"];

    var kişi = {

    ad: "John",

    soyAd: "Doe",

    yaş: 50,

    eyeColor: "mavi"

    };

    Karmaşık (bileşik) ifadeler için genel kurallar:

  • Açma braketini ilk satırın sonuna yerleştirin.

  • Açma braketinden önce bir boşluk kullanın.

  • Kapatma dirseğini, boşluk bırakmadan yeni bir hatta yerleştirin.

  • Noktalı virgülle karmaşık bir ifadeyi sonlandırmayın.

    • Fonksiyonlar:

    functionCelsius (fahrenheit) {

    dönüş (5/9) * (fahrenhayt - 32);

    }

    Döngüler

    for (i = 0; i <5; i ++) {

    x + = i;

    }

    Şartlılar:

    if (zaman <20) {

    tebrik = "İyi günler";

    } else {

    tebrik = "İyi akşamlar";

    }

    Nesne Kuralları

    Nesne tanımları için genel kurallar:

  • Açılış köşeli aygırı, nesne adıyla aynı satıra yerleştirin.

  • Her özellik ve değeri arasında iki nokta üst üste artı bir boşluk kullanın.

  • Sayısal değerlerin çevresinde değil, dize değerleri çevresinde tırnak kullanın.

  • Son özellik-değer çiftinden sonra virgül eklemeyin.

  • Kapatma dirseğini ön boşluk bırakmadan yeni bir satıra yerleştirin.

  • Nesne tanımını her zaman noktalı virgülle sonlandırın.

    • var kişi = {

    ad: "John",

    soyAd: "Doe",

    yaş: 50,

    gozRengi: "mavi"

    };

    Kısa nesneler, aşağıdaki gibi yalnızca özellikler arasındaki boşluklar kullanılarak bir satıra sıkıştırılmış olarak yazılabilir:

    var person = {firstName: "John", lastName: "Doe", yaş: 50, eyeColor: "blue"};

    Adlandırma Kuralları

    Tüm kodlarınız için her zaman aynı adlandırma kuralını kullanın. Örneğin:

  • CamelCase olarak yazılan değişken ve işlev adları

  • UPPERCASE ile yazılmış global değişkenler (Yapmıyoruz, ama oldukça yaygın)

  • UPPERCASE ile yazılmış sabitler (PI gibi)

    • Değişken adlarında hip-tavuk, camelCase veya under_scores kullanmalı mısınız?

    Bu, programcıların sıkça tartıştığı bir sorudur. Cevap, kime sorduğunuza bağlıdır:

    HTML ve CSS'deki tire işaretleri:

    HTML5 özellikleri verilerle başlayabilir - (veri miktarı, veri fiyatı).

    CSS, özellik adlarında (yazı tipi boyutu) kısa çizgiler kullanır.

    Kısa çizgiler çıkarma girişimleri olarak karıştırılabilir. JavaScript adlarında kısa çizgilere izin verilmez.

    UndersCores:

    Birçok programcı, özellikle SQL veritabanlarında alt çizgi (date_of_birth) kullanmayı tercih eder.

    Alt çizgiler genellikle PHP belgelerinde kullanılır.

    PascalCase:

    PascalCase genellikle C programcıları tarafından tercih edilir.

    CamelCase:

    CamelCase, JavaScript'in kendisi, jQuery ve diğer JavaScript kütüphaneleri tarafından kullanılır.

    $ İşaretiyle isimleri başlatmayın. Sizi birçok JavaScript kitaplığı adıyla çakıştıracaktır.

    HTML'ye JavaScript yükleme

    Harici komut dosyalarını yüklemek için basit bir sözdizimi kullanın (type özelliği gerekli değildir):

    < script src = "myscript.js"> < /script>

    HTML Öğelerine Erişme

    "Düzensiz" HTML stilleri kullanmanın bir sonucu olarak, JavaScript hataları oluşabilir.

    Bu iki JavaScript ifadesi farklı sonuçlar üretecektir:

    var obj = getElementById ("Demo")

    var obj = getElementById ("demo")

    Mümkünse, HTML'de aynı adlandırma kuralını (JavaScript olarak) kullanın.

    Dosya uzantıları

  • HTML dosyalarının .html uzantısı (.htm değil) olmalıdır.

  • CSS dosyalarının .css uzantısı olmalıdır.

  • JavaScript dosyalarının .js uzantısı olmalıdır.

    • Küçük Harfli Dosya Adlarını Kullan

    Çoğu web sunucusu (Apache, Unix) dosya adları hakkında büyük / küçük harfe duyarlıdır:

    london.jpg dosyasına London.jpg olarak erişilemez.

    Diğer web sunucuları (Microsoft, IIS) büyük / küçük harfe duyarlı değildir:

    london.jpg'ye London.jpg veya london.jpg olarak erişilebilir.

    Büyük ve küçük harflerden oluşan bir karışım kullanıyorsanız, son derece tutarlı olmanız gerekir.

    Duyarsız bir durumdan çıkarsanız, büyük / küçük harf duyarlı bir sunucuya, küçük hatalar bile web sitenizi kırabilir.

    Bu sorunları önlemek için her zaman küçük harfli dosya adları kullanın (mümkünse).

    Verim

    Kodlama kuralları bilgisayarlar tarafından kullanılmaz. Kuralların çoğunun programların yürütülmesi üzerinde çok az etkisi vardır.

    Girinti ve fazladan boşluklar küçük komut dosyalarında önemli değildir.

    Geliştirilmekte olan kod için okunabilirlik tercih edilmelidir. Daha büyük üretim senaryoları küçültülmelidir.
